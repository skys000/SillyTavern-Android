# SillyTavern Android

[中文文档](README_CN.md)

Self-contained Android app that runs [SillyTavern](https://github.com/SillyTavern/SillyTavern) natively — no Termux, no root, install and go.

## Features

- **Install and use** — no Termux, no root, no configuration
- **Embedded Node.js** — ARM64 runtime bundled in APK
- **Full SillyTavern** — complete server with all features
- **Native Android UX** — immersive WebView, back navigation, pull-to-refresh, file upload
- **Dark theme** — matches SillyTavern's aesthetic

## For Users: Download APK

Check [Releases](../../releases) for pre-built APKs. Requirements:
- Android 8.0+ (API 26)
- ARM64 device (99%+ of modern phones)
- ~500 MB free storage

## For Developers: Build From Source

Only **2 steps**. You need: a PC with [Android Studio](https://developer.android.google.cn/studio), Python 3, Git, and Node.js (npm).

### Step 1: Prepare assets

```bash
python scripts/setup_assets.py
```

This single command downloads everything automatically:
1. **Node.js ARM64 binary** from Termux package repository → `app/src/main/jniLibs/arm64-v8a/libnode.so`
2. **Shared libraries** (libz, libssl, libicu, etc.) → `app/src/main/assets/nodelibs.zip`
3. **SillyTavern** from GitHub + `npm install` → `app/src/main/assets/sillytavern.zip`

No Android phone needed — Node.js is fetched directly from Termux's public package server.

<details>
<summary>Advanced options</summary>

```bash
# Use a different SillyTavern branch
python scripts/setup_assets.py --st-branch staging

# Use a Termux export zip (from phone) instead of downloading
python scripts/setup_assets.py --from-termux path/to/node-android-arm64.zip
```

To create the Termux export zip, copy `scripts/termux_export.sh` to your phone and run it in Termux.
</details>

### Step 2: Build APK

1. Open this project in Android Studio
2. **Build** → **Build Bundle(s) / APK(s)** → **Build APK(s)**
3. Output: `app/build/outputs/apk/debug/app-debug.apk`
4. Install on phone

## How It Works

```
┌──────────────────────────────────┐
│          Android App             │
│                                  │
│  1. First launch:                │
│     Extract assets to internal   │
│     storage (sillytavern.zip     │
│     + nodelibs.zip)              │
│                                  │
│  2. Start Node.js process        │
│     (libnode.so from nativeLibs) │
│     LD_LIBRARY_PATH → nodelibs/  │
│     Runs: node server.js         │
│                                  │
│  3. SillyTavern HTTP server      │
│     listening on 127.0.0.1:8000  │
│                                  │
│  4. WebView loads localhost:8000  │
│     Full SillyTavern UI          │
└──────────────────────────────────┘
```

### Why Termux libraries?

Node.js is not officially supported on Android. The most reliable way to get a working ARM64 Node.js binary is from [Termux](https://termux.dev/). However, Termux's Node.js links against Termux's own shared libraries (libz, libssl, libicu, etc.) instead of Android's system libraries. We bundle these alongside the binary and set `LD_LIBRARY_PATH` at runtime.

## Project Structure

```
├── app/src/main/
│   ├── kotlin/.../
│   │   ├── LauncherActivity.kt  # Extract → start node → open WebView
│   │   ├── MainActivity.kt      # Full-screen WebView
│   │   ├── NodeManager.kt       # Node.js process & extraction lifecycle
│   │   ├── NodeService.kt       # Foreground service
│   │   └── SettingsActivity.kt  # Settings UI
│   ├── assets/                  # ← generated by setup_assets.py
│   ├── jniLibs/arm64-v8a/      # ← generated by setup_assets.py
│   ├── res/
│   └── AndroidManifest.xml
├── scripts/
│   ├── setup_assets.py          # One command: download + package everything
│   ├── download_node.py         # Download Node.js from Termux repo
│   ├── package_sillytavern.py   # Clone + package SillyTavern
│   ├── package_nodelibs.py      # Package .so libraries
│   └── termux_export.sh         # (optional) export from phone
└── README.md
```
